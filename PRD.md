# cc-router PRD — 产品需求文档

> 版本：1.0 | 日期：2026-05-27

## 1. 背景与问题

### 1.1 现状

开发者使用 Claude Code 作为编码辅助工具时，通常需要配置一个 Anthropic 兼容的 API 后端。DeepSeek V4 Pro 提供了 Anthropic Messages API 兼容端点，成本低、文本能力强，但**不支持图片输入**。

### 1.2 痛点

- Claude Code 工作流中经常需要截图报错、查看设计稿、分析图表，这些场景需要多模态能力
- DeepSeek V4 Pro 遇到图片请求直接报错。用户要么全程用贵的多模态模型（浪费），要么每次遇到图片手动改配置（打断工作流）
- 没有统一的地方查看请求路由情况和成本分布

### 1.3 目标用户

使用 Claude Code + DeepSeek V4 Pro（或类似仅文本模型）作为日常编码辅助的开发者。

## 2. 产品定义

### 2.1 一句话描述

cc-router 是一个轻量级本地代理，自动检测 Claude Code 请求中的图片内容，将多模态请求路由到支持视觉的模型，纯文本请求路由到高性价比模型，实现无感切换。

### 2.2 核心价值

| 价值 | 说明 |
|------|------|
| 省钱 | 纯文本走 DeepSeek V4 Pro 等低成本模型，不因偶尔用图就全程付高价 |
| 无感 | Claude Code 无需任何改动，正常使用即可，路由完全透明 |
| 灵活 | 多模态后端可配置，支持 OpenRouter、直连等多种方式 |
| 轻量 | 单文件 Python，无数据库依赖，开箱即用 |

## 3. 功能需求

### 3.1 P0 — 核心功能

| 编号 | 功能 | 描述 |
|------|------|------|
| F01 | 图片检测 | 自动解析请求体，识别 `type: "image"` 内容块 |
| F02 | 智能路由 | 有图片 → 多模态后端；无图片 → 纯文本后端 |
| F03 | 请求转发 | 完整透传请求体和响应体，替换模型名和 API 密钥 |
| F04 | 流式支持 | 支持 SSE 流式转发，保证 Claude Code 打字体验 |
| F05 | 双后端配置 | 支持配置纯文本后端和多模态后端的 URL、密钥、模型名 |

### 3.2 P1 — 重要功能

| 编号 | 功能 | 描述 |
|------|------|------|
| F06 | 配置热重载 | 修改配置文件后无需重启，下次请求自动生效 |
| F07 | 路由日志 | 每次请求输出路由决策（类型、源模型 → 目标模型） |
| F08 | 状态页面 | Web 页面展示当前配置、运行时间、路由统计 |
| F09 | 健康检查 | `/health` 端点供外部监控 |

### 3.3 P2 — 锦上添花

| 编号 | 功能 | 描述 |
|------|------|------|
| F10 | 路由规则扩展 | 支持按 token 数、关键词等更多维度路由 |
| F11 | 多后端负载均衡 | 同类请求支持配置多个后端轮询/随机 |
| F12 | 用量统计持久化 | 路由统计写入 SQLite，重启不丢失 |
| F13 | 认证中间件 | 代理本身支持 API Key 认证，防止本地其他程序滥用 |

## 4. 用户场景

### 场景 A：日常编码（纯文本）

1. 用户在终端运行 Claude Code，输入"帮我重构这个函数"
2. cc-router 检测无图片，路由到 DeepSeek V4 Pro
3. 响应正常返回，用户无感知

### 场景 B：截图报错（多模态）

1. 用户在 Claude Code 中粘贴一张报错截图
2. cc-router 检测到 `type: "image"`，路由到多模态后端
3. 多模态模型分析截图并返回结果
4. Claude Code 正常展示

### 场景 C：切换多模态后端

1. 用户觉得 Qwen3.7 看图不够准，想换 Claude Sonnet 4
2. 编辑 config.yaml，修改 multimodal 模型配置
3. 无需重启代理，下次带图请求自动走新后端
4. 打开 /status 页面确认配置已更新

### 场景 D：检查成本

1. 用户想了解自己多少请求走了多模态、多少走了纯文本
2. 打开 http://127.0.0.1:8082/status
3. 查看路由统计数据

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 代理转发延迟 < 50ms（不含后端响应时间） |
| 可靠性 | 代理崩溃不影响 Claude Code 本身，重新启动即可恢复 |
| 安全性 | API Key 仅存在本地配置文件，不外传；仅监听 localhost |
| 兼容性 | 支持 Python 3.10+；兼容 macOS / Linux |
| 部署 | 零外部依赖服务（无 Redis/MySQL 等），pip install 即可运行 |

## 6. 约束与假设

- **约束 1**：后端必须兼容 Anthropic Messages API 格式，代理不做格式转换
- **约束 2**：仅支持 Claude Code 场景，不保证兼容其他 Anthropic API 客户端
- **假设 1**：用户已在本地运行代理，Claude Code 通过环境变量指向代理地址
- **假设 2**：用户已拥有 DeepSeek 和至少一个多模态后端的 API Key

## 7. 成功指标

| 指标 | 目标 |
|------|------|
| 路由准确率 | 图片请求 100% 路由到多模态，纯文本请求 0% 误路由到多模态 |
| 代理可用性 | 运行期间请求转发成功率 > 99.9%（不含后端自身故障） |
| 用户感知 | 使用代理后，Claude Code 功能完整性无降低 |
